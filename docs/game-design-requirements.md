# 9ronTactics ゲーム性の修正要件

## このゲームの前提

9ronTactics は、元々ボードゲームとして成立している対戦ゲームである。

プレイヤーは数字の書かれたコマを持ち、各ラウンドでコマを裏向きに出す。数字の大小によって勝敗は決まるが、相手が実際に何の数字を出したのかは最後までわからない。

重要なのは、数字そのものではなく、限られた公開情報から相手の手を読むことである。

## 公開される情報

プレイヤーに見えてよい情報は次の通り。

- 自分が出したコマの数字
- 相手が出したコマの色
- ラウンドの勝敗
- 相手の残りコマの色の状況

コマの色は偶数・奇数を表す。

- 赤: 奇数
- 青: 偶数

つまり、相手が出した数字そのものは見えないが、奇数を出したのか偶数を出したのかはわかる。

## 隠される情報

プレイヤーに見えてはいけない情報は次の通り。

- 相手が出したコマの数字
- 相手の残りコマの具体的な数字
- ラウンド終了後の相手の使用済み数字

現行実装では、ラウンド終了後の履歴に相手の数字が表示されてしまっている。これはゲーム性を壊しているため修正対象とする。

## 必要なUI

### ラウンド中の表示

相手がコマを出したら、相手のコマの数字ではなく色だけが見える必要がある。

- 相手が奇数を出した場合: 赤い裏向きコマとして表示
- 相手が偶数を出した場合: 青い裏向きコマとして表示
- 数字は表示しない

自分のコマは、自分が選んだ数字を表示してよい。

### ラウンド結果の表示

ラウンド終了後に表示してよいのは勝敗だけである。

- 勝ち
- 負け
- 引き分け

相手が何番を出して勝ったのか、または負けたのかは表示しない。

履歴にも相手の数字は表示しない。履歴に残すなら、相手の色と勝敗だけにする。

### 相手の残りコマ表示

相手の残りコマについて、具体的な数字は表示しない。

ただし、色の状況は表示する。

- 赤いコマが何枚残っているか
- 青いコマが何枚残っているか

これにより、プレイヤーは相手の残り数字を直接知るのではなく、奇数・偶数の偏りから推測する。

## 今回直したいこと

今回の修正対象は、ゲームルール全体の再設計ではない。

直したいのは、現行実装が本来隠すべき情報を見せてしまっている点と、本来見えるべき色の情報を見せていない点である。

優先して直す内容は次の3つ。

1. 相手が出したコマは数字ではなく色だけ見せる
2. ラウンド履歴やリザルトに相手の数字を表示しない
3. 相手の残りコマを数字ではなく赤/青の枚数で表示する

この3点が、このゲームの読み合いを成立させるために必要な最小要件である。
